# RPLU Bring-Up Guard

This guard keeps the Tang Primer 25K RPLU flash-loaded path repeatable after
the first hardware proof.

## Software Regression

Run the focused regression before changing the RPLU loader, RPLU tables, or
SPU-13 axis normalization:

```sh
tools/run_rplu_bringup_regression.sh
```

The metric/reference step uses only Python's standard library; no virtualenv is
required for this guard.

It rebuilds the RPLU SPI payload and runs:

- `rplu_metric_reference.py`: source-table metrics, generated RTL vectors,
  and flash chord payload CRC/match.
- `davis_to_rplu_tb`: large-axis clamp and delayed-start address capture.
- `rplu_exp_tb`: RPLU vnorm/dissoc table behavior.
- `rplu_metric_vectors_tb`: RTL table lookup against generated metric vectors.
- `spu_rotor_vault_tb`: Pell rotor vault reset, hydration-safe storage, and
  per-axis octave walk used by the RPLU+math datapath probe.
- `spu13_rplu_addr_tb`: SPU-13 axis walk reaches RPLU address `0x3FF`.
- `spu_laminar_boot_rplu_tb`: SPI bootloader emits RPLU config writes from
  chord records read at `0x110000`.

To preserve an existing 16 MiB flash image while rebuilding the RPLU regions:

```sh
RPLU_BASE_IMAGE=build/j4_flash_backup_before_rplu.bin tools/run_rplu_bringup_regression.sh
```

## Flash Image

Build a standalone image:

```sh
python3 tools/build_tang25k_j4_rplu_flash.py
```

The standalone image fills bytes outside the SPU/RPLU regions with `0xFF`.
For a board flash that also carries other boot/config contents, program the
merged image instead.

Build a merged image that preserves the original flash contents outside the
Pell, golden-prime, and RPLU regions:

```sh
python3 tools/build_tang25k_j4_rplu_flash.py \
  --base-image build/j4_flash_backup_before_rplu.bin \
  --output build/tang25k_j4_rplu_flash_merged.bin
```

## Hardware Probe

With the verified SPI flash installed and the FPGA connected:

```sh
tools/probe_tang25k_rplu_flash.py
```

The probe loads `build/tang_primer_25k_spu13_rplu_probe.fs`, listens on the
Sipeed FTDI UART at 115200 baud, and waits up to 45 seconds. The timeout covers
the repeated boot-summary window; capture exits early once the JEDEC, RPLU
marker, loaded-count, and checksum lines have all been observed. By default,
the expected loaded count and checksum are read from the generated RPLU metric
summary/payload, so the probe follows the flash image that the regression just
built.

The proof line is:

```text
R:D28003FF A:D
```

Decode:

- `marker = 0x1A5`: at least one RPLU config chord was loaded from flash.
- `mask = 0x0000`: RPLU dissociation mask telemetry for the current probe.
- `addr = 0x3FF`: the large-radius path reaches the high end of the table.

The stronger proof lines are:

```text
R:00000803 A:E
R:<checksum> A:F
```

- `A:E`: loaded RPLU record count, currently `2051` (`0x0803`).
- `A:F`: cheap RTL rolling checksum over loaded RPLU chord records. The
  expected value is generated by `tools/rplu_metric_reference.py` in
  `build/rplu_metrics/rplu_metric_summary.json`.

The expected SPI flash ID line is:

```text
B:D0EF4018 A:C
```

## Datapath Probe

After the RPLU-only proof passes, build the next-stage image that keeps SDRAM
and lattice disabled but enables both flash-loaded RPLU and the SPU-13
rotor/Davis math path:

```sh
./build_25k_spu13_rplu_math_probe.sh
tools/probe_tang25k_rplu_flash.py \
  --bitstream build/tang_primer_25k_spu13_rplu_math_probe.fs
```

This uses the same UART proof lines, but now `A:D` comes from the RPLU lookup
path driven by the live rotated SPU-13 axis data.

The next composition step keeps lattice disabled but adds the external SDRAM
bridge and manifold writeback:

```sh
./build_25k_spu13_rplu_math_sdram_probe.sh
tools/probe_tang25k_rplu_flash.py \
  --bitstream build/tang_primer_25k_spu13_rplu_math_sdram_probe.fs
```
